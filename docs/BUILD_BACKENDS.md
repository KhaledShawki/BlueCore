# Blue Build Backends

Blue uses Premake as the single source of truth for project structure. Generated build files and binaries must stay under `out/`.

## Backend policy

| Environment | Primary backend | Notes |
| --- | --- | --- |
| Windows development | Visual Studio (`vs2026` / `vs2022`) | Full solution helper targets and static/shared matrix. |
| macOS development | Ninja | Fast local builds and test runs. Static `x64` variant by default. |
| Linux development | Ninja | Fast local builds and CI. Static `x64` variant by default. |
| Unix fallback | `gmake2` | Use when diagnosing Ninja generator/toolchain issues. |
| Tooling | `compilecommands` | Use for clangd/CLion/VS Code code intelligence. |

## Ninja policy

Premake's Ninja backend does not handle Blue's current per-platform static/shared kind switching in a single project file. Therefore Blue generates only the static `x64` build platform for Ninja by default.

Use Visual Studio or `gmake2` for full `x64` + `x64_DLL` matrix builds until the shared-library Ninja model is split into separate static/shared targets.

## Commands

macOS:

```bash
./scripts/run-tests-macos.sh Debug_x64
```

Linux:

```bash
./scripts/run-tests-linux.sh Debug_x64
```

Manual generation:

```bash
./scripts/premake-macos.sh ninja --toolchain=clang --blue-platforms=macos --blue-build-platforms=x64 --memory-backend=system --blue-startup=BlueRunTests
ninja -C out/build/ninja BlueRunTests_Debug_x64
```

## Output directories

Generated files and binaries belong under:

```text
out/build/<backend>
out/bin/<system>/<platform>/<config>
out/obj/<system>/<platform>/<config>
```

Nothing generated by the build should be written into `modules/`, `apps/`, `tests/`, or `build/`.


## macOS SDK handling

Blue's Ninja macOS flow uses Apple Clang through `xcrun` and injects the active macOS SDK into generated compile and link commands. This avoids accidental use of a Homebrew or incomplete `clang++` installation that cannot see headers such as `pthread.h`, `errno.h`, `time.h`, or libc++ headers such as `type_traits`.

The test script creates temporary compiler wrappers under `out/tools/macos-clang/` and prepends that directory to `PATH` before invoking Ninja. The wrappers are generated artifacts and may be deleted with the rest of `out/`.
